# ファーストビュー画像

このディレクトリには、LPトップに表示されるファーストビュー画像を配置します。

## 画像の配置方法

1. このディレクトリに **`hero.jpg`** というファイル名で画像をアップロードしてください
2. アップロードと同時に、自動でLPのファーストビューに表示されます
3. HTMLやCSSを編集する必要はありません

### GitHub Web画面からアップロードする手順

1. GitHub のリポジトリページで `assets/img/fv/` ディレクトリを開く
2. 右上の「Add file」→「Upload files」をクリック
3. お手持ちの画像ファイルをドラッグ＆ドロップ
4. **ファイル名が `hero.jpg` であることを確認**（違う場合はリネーム）
5. 「Commit changes」をクリック

## 推奨画像仕様

| 項目 | 推奨 |
|---|---|
| ファイル名 | `hero.jpg` |
| 形式 | JPG / WebP（WebPの場合はHTML側もパスを変更してください） |
| アスペクト比 | **3:2 推奨**（1500×1000px 等） |
| 推奨幅 | 1500〜1920px |
| ファイルサイズ | **300KB以下推奨**（表示速度のため） |
| 横長配置 | 横位置（SP表示でも上下端が切れないよう構図に余裕を持たせる） |

## 仕様詳細

- 画像全体が **予約フォーム（`#reserve`）** へのクリックリンクになっています
- 画像が見つからない場合は `placeholder.svg`（プレースホルダー）がフォールバック表示されます
- 画像読み込み中のCLS（レイアウトずれ）を防ぐため、CSS側で `aspect-ratio: 3 / 2` を指定済み
- もし 16:9 や 4:3 等の別アスペクト比を使う場合は、`assets/css/main.css` 内 `.fv__image` の `aspect-ratio` 値を変更してください

## 別の画像形式・ファイル名を使いたい場合

`hero.jpg` 以外を使う場合は、`index.html` 内の以下を編集してください：

```html
<img
  src="assets/img/fv/hero.jpg"  ← ここを変更
  ...
  onerror="this.onerror=null;this.src='assets/img/fv/placeholder.svg';">
```
